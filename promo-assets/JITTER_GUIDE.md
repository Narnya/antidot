## Развилка: как Jitter «думает»

Jitter — это не After Effects со слоями и ключевыми кадрами на каждое свойство. Модель проще, ближе к «Magic Move» из Keynote:

- Проект состоит из **Frames** (кадров-состояний) на таймлайне слева направо.
- Переход **между** двумя соседними Frames Jitter анимирует автоматически: если объект есть в обоих и сместился/поменял прозрачность — Jitter плавно его «перетекает».
- Плюс есть **Animate-пресеты** на каждый элемент (entrance/exit: Fade, Slide, Scale, Pop) — это для появления/исчезновения.

Из этого следует: чтобы карточка «выехала снизу», нужно либо чтобы она была **отдельным слоем** (тогда вешаешь на неё Slide-in пресет), либо два Frame — «до» (карточка за экраном) и «после» (на месте).

**Отдельные слои появляются только при импорте из Figma-плагина.** PNG — это плоская картинка, слоёв в ней нет.

## Метод A — Figma-плагин (РЕКОМЕНДУЕТСЯ)

Даёт object-level моушн: карточки выезжают по одной, аватары каскадятся, бейджи всплывают.

### Шаг 0. Подготовка
1. Зарегистрируйся на [jitter.video](https://jitter.video), войди.
2. Нужен Figma-аккаунт с доступом к файлу прототипа.

### Шаг 1. Установить связь с Figma
1. В Jitter: левое меню → **Resources / Import** → **Figma**.
2. Авторизуй доступ к Figma (откроется окно OAuth).
3. ИЛИ установи плагин «Jitter» прямо в Figma (Figma → Plugins → найти «Jitter» → Run), и оттуда отправляй кадры в Jitter.

### Шаг 2. Импортировать ВСЕ 6 кадров разом
1. Открой в Figma файл прототипа.
2. На странице `01 · Foundations & Components` найди борд `03 — Promo Storyboard v1`.
3. Зажми Shift и выдели **6 phone-фреймов** сразу:

   | Кадр | Имя слоя | Node ID |
   |------|----------|---------|
   | 01 | `Kadr 01` | `140:19` |
   | 02 | `Kadr 02` | `140:32` |
   | 03 | `Kadr 03` | `140:60` |
   | 04 | `Kadr 04` | `141:8` |
   | 05 | `Kadr 05` | `141:29` |
   | 06 | `Kadr 06` | `141:72` |

4. В плагине Jitter нажми **Import selection** (или в Jitter-вебе вставь ссылку на фрейм).
5. Все 6 приедут в Jitter, **каждый со своими слоями внутри** (фон, текст, карточки, аватары — отдельно).

> **Почему разом, а не по одному:** так Jitter сложит их на один таймлайн в правильном порядке, и тебе не придётся согласовывать размеры/позиции между отдельными импортами. Один импорт = один консистентный проект.

### Шаг 3. Настроить композицию
1. Canvas size: **1080 × 1920** (9:16).
2. FPS: **30**.
3. Если кадры приехали как 270×480 — выдели все, отмасштабируй до ширины канваса (Jitter держит вектор/растр чётким).

### Шаг 4. Разложить кадры по таймлайну
Jitter положит 6 кадров либо как 6 Frames, либо стопкой на одном канвасе — зависит от версии. Цель: получить **6 Frames на таймлайне** в порядке 01→06.

- Если приехали стопкой: вынеси каждый кадр в свой Frame (дублируй Frame, оставь в нём только нужный кадр).
- Выстави длительность каждого Frame по script:

  | Frame | Кадр | Длительность |
  |-------|------|-------------|
  | 1 | Хук | 4.0 с |
  | 2 | Альтернатива | 5.0 с |
  | 3 | Подача→Одобрение | 4.0 с |
  | 4 | Встреча в реале | 4.0 с |
  | 5 | Связи | 3.0 с |
  | 6 | Бренд | 3.0 с |

  **Итого 23 секунды.**

### Шаг 5. Анимировать ПО ОДНОМУ кадру
Открой [PROMO_VIDEO_SCRIPT_v1.md → Motion timing](../docs/PROMO_VIDEO_SCRIPT_v1.md#motion-timing--jitter-пресеты) рядом. Идёшь сверху вниз, кадр за кадром.

Для каждого элемента в кадре:
1. Выдели слой на канвасе.
2. Правая панель → **Animate** → выбери entrance-пресет (Fade / Slide / Scale / Pop).
3. На таймлайне сдвинь начало анимации (Start) и длину (Duration) по таблице кадра.
4. Easing: **Out Cubic** для входов, **In-Out Cubic** для движений камеры. Никаких bounce/elastic.

**Конкретно по кадрам (из script):**
- **Хук:** фон Fade In → текст Fade+Slide Up через 0.6с → коралловая точка Scale In через 1.2с → hold → Fade Out на 3.4с.
- **Альтернатива:** заголовок Fade+Slide Down → 3 карточки Slide Up **со сдвигом 0.3с между ними** (это и есть «каскад») → лёгкий зум всего кадра.
- **Подача:** обложка Scale 0.95→1.0 → текст → зелёный banner Slide Up через 1.0с → ✓ Scale In → кнопка Fade.
- **Встреча:** фото-карточка Ken Burns зум 1.0→1.06 на все 4с → текст Fade+Slide Up.
- **Связи:** заголовок → 4 аватара Slide In Left **со сдвигом 0.2с** → ✓-бейджи Scale In каскадом.
- **Бренд:** точка Pulse (loop x2) → wordmark Fade+Slide Up → домен Fade → pill Slide Up.

> **Правило:** доделал кадр — нажми Play, проверь только его, потом переходи к следующему. Не пытайся отполировать все 6 сразу.

### Шаг 6. Переходы между кадрами
Между Frames поставь **cross-fade 200 мс**. В Jitter это либо свойство перехода Frame (Transition: Dissolve 0.2s), либо вручную — последний слой кадра N Fade Out + первый слой кадра N+1 Fade In с нахлёстом 0.2с.

### Шаг 7. Превью и экспорт
1. Play на всю композицию — смотри ритм целиком.
2. Подгони тайминги, где «провисает» или «частит».
3. **Export** → MP4 → 1080×1920, 30 fps, High quality.
4. Сохрани как `promo-assets/jitter_export.mp4`.

## Метод B — PNG-импорт (быстро, но без object-level)

Если плагин не ставится или нет Pro-Figma.

1. Jitter → New project → Import → выбери все 6 PNG из `jitter-export/`.
2. Canvas 1080×1920, 30 fps.
3. Каждый PNG → свой Frame, длительности те же (4/5/4/4/3/3 с).
4. Анимировать можно только **кадр целиком**: Fade между кадрами + Scale (Ken Burns) на каждый.
5. Это ровно то, что уже делает `preview.html` — так что Метод B имеет смысл только если хочешь собрать всё в одном Jitter-проекте с экспортом в MP4 без скринкаста.

## Частые затыки

| Симптом | Причина | Что делать |
|---------|---------|-----------|
| Карточки нельзя анимировать по одной | Импортнул PNG, не из Figma | Переимпортируй через Figma-плагин (Метод A) |
| Текст «прыгает» при переходе | Слой с одним именем в двух Frames с разными свойствами — Jitter его твинит | Переименуй слой или убери из одного Frame |
| Анимация дёргается | Слишком много тяжёлых эффектов / низкий FPS | Поставь 30 fps, упрости easing |
| Кадр 04 (фото) статичный и «мёртвый» | Только Ken Burns мало | Прогони фото через Luma image-to-video, подложи клип |
| Шрифты не те | Inter не подгрузился | В Jitter выбери Inter вручную, или импортируй кадр как изображение |

## Что Jitter НЕ делает (и куда идти)

- **Звук** — Jitter без аудио. Войсовер + музыку клади в CapCut/DaVinci поверх экспортнутого MP4.
- **Сложный 3D / частицы** — не его профиль, и тебе не нужно.
- **Живое видео людей** — только Кадр 04 через Luma/Runway, отдельно.
